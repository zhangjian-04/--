# BigEvent 用户管理模块

## 项目概述

BigEvent 是一个基于 Spring Boot 的用户管理系统，提供了完整的用户注册、登录、信息管理等功能。该项目使用 JWT 进行身份认证，Redis 进行会话管理，并集成了阿里云 OSS 进行文件上传。

## 技术栈

- **后端框架**: Spring Boot
- **数据库**: MySQL (通过 MyBatis 进行数据访问)
- **缓存**: Redis
- **身份认证**: JWT (JSON Web Token)
- **文件存储**: 阿里云 OSS
- **密码加密**: MD5
- **参数验证**: Hibernate Validator

## 用户控制器 (UserController) API 文档

### 基础信息

- **控制器路径**: `/user`
- **验证**: 使用 `@Validated` 注解进行参数验证
- **返回格式**: 统一使用 `Result<T>` 包装响应数据

### API 接口列表

#### 1. 用户注册

- **接口路径**: `POST /user/register`
- **功能描述**: 注册新用户
- **请求参数**:
  - `username`: 用户名 (5-16 位非空白字符)
  - `password`: 密码 (5-16 位非空白字符)
- **参数验证**: 使用 `@Pattern(regexp = "^\\S{5,16}$")` 验证用户名和密码格式
- **业务逻辑**:
  - 检查用户名是否已存在
  - 如果不存在则创建新用户
  - 返回注册结果
- **返回示例**:
  ```json
  {
    "code": 200,
    "msg": "操作成功",
    "data": null
  }
  ```

#### 2. 用户登录

- **接口路径**: `POST /user/login`
- **功能描述**: 用户登录认证
- **请求参数**:
  - `username`: 用户名 (5-16 位非空白字符)
  - `password`: 密码 (5-16 位非空白字符)
- **业务逻辑**:
  - 验证用户名是否存在
  - 验证密码是否正确 (MD5 加密比较)
  - 生成 JWT Token
  - 将 Token 存储到 Redis (有效期 1 小时)
- **返回示例**:
  ```json
  {
    "code": 200,
    "msg": "操作成功",
    "data": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
  ```

#### 3. 获取用户信息

- **接口路径**: `GET /user/userInfo`
- **功能描述**: 根据 Token 获取当前用户信息
- **认证方式**: 通过 ThreadLocal 获取用户信息 (需要登录拦截器支持)
- **业务逻辑**:
  - 从 ThreadLocal 中获取用户名
  - 查询用户详细信息
- **返回示例**:
  ```json
  {
    "code": 200,
    "msg": "操作成功",
    "data": {
      "id": 1,
      "username": "testuser",
      "password": "encrypted_password",
      "nickname": "测试用户",
      "email": "test@example.com",
      "userPic": "avatar_url"
    }
  }
  ```

#### 4. 更新用户信息

- **接口路径**: `PUT /user/update`
- **功能描述**: 更新用户基本信息
- **请求体**: JSON 格式的用户对象
- **参数验证**: 使用 `@Validated` 和实体类验证注解
- **业务逻辑**:
  - 验证用户数据格式
  - 更新用户信息到数据库
- **返回示例**:
  ```json
  {
    "code": 200,
    "msg": "操作成功",
    "data": null
  }
  ```

#### 5. 更新用户头像

- **接口路径**: `PATCH /user/updateAvatar`
- **功能描述**: 更新用户头像
- **请求参数**:
  - `avatarUrl`: 头像 URL (使用 `@URL` 验证格式)
- **业务逻辑**:
  - 验证 URL 格式
  - 更新用户头像 URL
- **返回示例**:
  ```json
  {
    "code": 200,
    "msg": "操作成功",
    "data": null
  }
  ```

#### 6. 修改密码

- **接口路径**: `PATCH /user/updatePwd`
- **功能描述**: 修改用户密码
- **请求头**: `Authorization: Bearer <token>`
- **请求体**:
  ```json
  {
    "old_pwd": "原密码",
    "new_pwd": "新密码",
    "re_pwd": "确认新密码"
  }
  ```
- **业务逻辑**:
  - 验证原密码是否正确
  - 验证两次新密码输入是否一致
  - 更新密码 (MD5 加密)
  - 删除 Redis 中的 Token (强制重新登录)
- **返回示例**:
  ```json
  {
    "code": 200,
    "msg": "操作成功",
    "data": null
  }
  ```

## 核心组件说明

### 1. 身份认证机制

- 使用 JWT Token 进行身份认证
- Token 存储在 Redis 中，设置 1 小时过期时间
- 通过 `ThreadLocalUtil` 在请求处理过程中传递用户信息

### 2. 密码安全

- 使用 MD5 对密码进行加密存储
- 修改密码后自动清除 Redis 中的 Token，强制重新登录

### 3. 参数验证

- 使用 Hibernate Validator 进行参数验证
- 支持正则表达式验证用户名和密码格式
- 支持 URL 格式验证头像链接

### 4. 异常处理

- 统一的错误响应格式
- 详细的错误信息提示

## 错误码说明

| 错误信息                                  | 说明                       |
| ----------------------------------------- | -------------------------- |
| "用户名已经存在了,请更换一个用户名！！！" | 注册时用户名已存在         |
| "该用户不存在"                            | 登录时用户名不存在         |
| "密码输入错误"                            | 登录时密码不正确           |
| "缺少必要的参数"                          | 修改密码时参数不完整       |
| "原密码输入不正确!"                       | 修改密码时原密码错误       |
| "两次密码输入不一致！"                    | 修改密码时两次新密码不一致 |

## 使用说明

1. **注册用户**: 调用注册接口创建新用户
2. **用户登录**: 使用用户名密码登录获取 Token
3. **访问受保护接口**: 在请求头中携带 Token
4. **更新信息**: 使用相应的更新接口修改用户信息
5. **修改密码**: 提供原密码和新密码进行密码修改

## 注意事项

- 所有密码都经过 MD5 加密处理
- Token 有效期为 1 小时，过期需要重新登录
- 修改密码后需要重新登录
- 用户名和密码必须为 5-16 位非空白字符
- 头像 URL 必须符合 URL 格式规范
